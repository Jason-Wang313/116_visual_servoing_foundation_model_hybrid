\begin{abstract}
Foundation policies can propose semantically appropriate robot actions, but local visual servo loops can be safer when the error is geometric and action-critical. The question is not which controller wins globally, but when a classical servo loop should override a learned action decoder. We study a calibrated visual-servo/foundation-policy arbiter in a local benchmark with 5 visual manipulation tasks, 7 visual/geometry shift regimes, 5 deployment splits, 9 controllers, 7 paired seeds, and 84 rollout episodes per group. Under combined stress, the proposed arbiter reaches $0.698 \pm 0.008$ success versus $0.598 \pm 0.008$ for the strongest non-oracle baseline, ensemble risk gating. The paired gain is $0.100 \pm 0.007$ with wins on 7/7 seeds. The arbiter also improves override precision, reduces unsafe learned actions, lowers tracking error, damage, and latency/control cost, and survives ablations. This is strong local evidence for a submission rebuild, but not final ICLR-main readiness without real robot or external high-fidelity validation.
\end{abstract}

\section{Motivation}

Classical visual servoing and robot foundation policies fail differently. Visual servoing is locally stabilizing but can oscillate under poor geometry, occlusion, or degenerate Jacobians. Foundation policies reason over semantics but may issue actions that are locally unsafe when visual error is large. A robust robot should arbitrate between them.

The claim is narrow: a hybrid controller should switch using action-critical visual error, geometry uncertainty, learned-action risk, servo stability margin, and latency cost. It should not blindly shield every learned action or always prefer visual servoing.

\section{Arbiter}

For state $x$ and candidate foundation action $a_f$, the arbiter computes
\[
A(x,a_f) =
\sigma\left(
\alpha e_{\mathrm{crit}}(x,a_f)
+ \beta u_{\mathrm{geom}}(x)
+ \gamma r_{\mathrm{learned}}(x,a_f)
- \delta m_{\mathrm{servo}}(x)
- \rho c_{\mathrm{latency}}(x)
\right),
\]
where $e_{\mathrm{crit}}$ is action-critical image error, $u_{\mathrm{geom}}$ is geometry uncertainty, $r_{\mathrm{learned}}$ is learned-action risk, $m_{\mathrm{servo}}$ is a servo stability margin, and $c_{\mathrm{latency}}$ penalizes costly switching. The controller chooses servo correction when $A$ exceeds a calibrated threshold, otherwise it follows the foundation action.

\section{Benchmark}

The benchmark includes eye-in-hand peg insertion, visual grasp centering, drawer-handle alignment, cable-tip alignment, and mobile pick re-approach. Regimes include nominal view, lighting shift, camera extrinsic drift, depth scale error, occlusion, Jacobian mismatch, and compound visual/geometry shift.

Baselines include foundation policy only, classical IBVS, position-based visual servoing, fixed hybrid switching, uncertainty-gated foundation control, CBF safety shielding, ensemble risk gating, the proposed arbiter, and an oracle arbiter.

\section{Related Work Boundary}

Image-based visual servoing, position-based visual servoing, sliding-mode visual control, hybrid visual control, and learned visual policies are all crowded areas \citep{ibvs2022,hybridvisual2025,hybridgrasp2009,pickingservo2019}. This paper does not claim new visual servoing theory. Its boundary is calibrated arbitration between learned foundation actions and classical servo corrections under visual and geometry shift.

\section{Limitations And Submission Decision}

The evidence is local and generated by the released benchmark. It is stronger than the archived scaffold but still lacks real robot experiments, external high-fidelity simulator transfer, released controller artifacts, and hardware videos. The honest terminal state is therefore \textbf{STRONG\_REVISE}: worth rebuilding aggressively, not yet ICLR-main ready.
